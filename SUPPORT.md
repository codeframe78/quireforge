# Support

QuireForge is pre-release software. There is currently no installable desktop
application, production website, package, or supported end-user release.

## Where to ask

- Bugs and reproducible documentation problems: use a repository bug report.
- Feature proposals: use a feature request and relate it to the roadmap.
- Build or contribution questions: use an issue with sanitized environment
  information.
- Security vulnerabilities: follow [SECURITY.md](SECURITY.md) and do not post
  sensitive details publicly.
- Codex account, billing, workspace policy, or official-product support:
  contact OpenAI through its official support channels. QuireForge cannot
  change account entitlements or administrator policy.
- Third-party connector or MCP incidents: contact the relevant provider as
  appropriate and revoke compromised credentials with their owner.

## Useful diagnostic information

When applicable, include:

- QuireForge revision or release version;
- Linux distribution and version;
- Wayland or X11 session;
- installed Codex and Git versions;
- sanitized reproduction steps and error text; and
- whether the directory is local, removable, network-backed, a symlink, or a
  Git worktree.

Remove usernames, home paths, repository content, access tokens, OAuth codes,
private URLs, account identifiers, and connector data. Do not upload a raw
QuireForge database, Codex directory, SSH configuration, environment file, or
browser profile.

## Project expectations

Support is community-provided on a best-effort basis. The roadmap is not a
release promise, and documented compatibility remains subject to installed
Codex capabilities, Linux packaging constraints, account entitlements, and
workspace policy.
